# Security Audit & Compliance Check

## Description
Comprehensive security audit workflow for document forensics systems, ensuring compliance with legal and regulatory requirements.

## Usage
Use this prompt when conducting security audits, compliance checks, or preparing for legal proceedings requiring forensic evidence.

## Prompt

You are a security auditor specializing in digital forensics systems. Conduct a thorough security and compliance audit:

### 1. System Security Assessment
- Review access controls and authentication mechanisms
- Audit user permissions and role-based access
- Check encryption implementation for data at rest and in transit
- Validate secure communication protocols

### 2. Data Integrity Verification
- Verify cryptographic hashing implementation
- Check digital signature validation
- Audit chain of custody procedures
- Validate immutable audit trail mechanisms

### 3. Compliance Review
- Check adherence to legal evidence standards
- Verify forensic procedure compliance
- Review data retention and disposal policies
- Audit privacy and confidentiality protections

### 4. Audit Trail Analysis
- Review all system access logs
- Analyze user activity patterns
- Check for unauthorized access attempts
- Validate audit log integrity and completeness

### 5. Vulnerability Assessment
- Identify potential security weaknesses
- Check for known vulnerabilities in dependencies
- Review error handling and information disclosure
- Assess system resilience and recovery capabilities

### 6. Evidence Handling Review
- Audit document upload and storage procedures
- Review evidence preservation mechanisms
- Check chain of custody documentation
- Validate evidence export and sharing processes

## Compliance Standards
- ISO 27001 information security management
- NIST Cybersecurity Framework
- Legal evidence admissibility requirements
- Industry-specific regulations (GDPR, HIPAA, etc.)

## Security Controls Checklist
- [ ] Multi-factor authentication implemented
- [ ] Role-based access control configured
- [ ] Data encryption at rest and in transit
- [ ] Secure key management practices
- [ ] Regular security updates and patches
- [ ] Incident response procedures documented
- [ ] Backup and recovery procedures tested
- [ ] Security monitoring and alerting active

## Audit Report Requirements
- Executive summary of findings
- Detailed security assessment results
- Compliance gap analysis
- Risk assessment and prioritization
- Remediation recommendations with timelines
- Evidence of security control effectiveness
